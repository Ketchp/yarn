Detailed Description

Ideal for implementing monitor synchronisation.

Note
No benefit in using yarn::Condition over pthread cond_t.
Use note, when checking or setting control variables that controls wake up of threads or signaling, lock must be acquired to prevent race conditions.
Single lock can be used with multiple Conditions (to minimise unnecessary wake-ups).
